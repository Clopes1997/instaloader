Privacy Policy

Instaloader · Last updated May 2026

🛡️

Short version: Instaloader collects no personal data, sends nothing to any server, and operates entirely within your browser. There is nothing to track, sell, or share.

Data collection

Instaloader does not collect, store, transmit, or share any personal information, browsing history, or usage data of any kind. No analytics, no telemetry, no crash reporting.

How it works

When you open the extension popup, it reads the media elements (images and videos) that are already loaded in your current browser tab. This processing happens entirely on your device. Nothing is sent to any external server — not to the developer, not to any third party.

Downloaded files are saved directly to your local machine via the browser's built-in download manager, following the same path as any normal file download.

Permissions explained

activeTab

Allows the extension to access the tab you are currently viewing when you click the icon. It cannot access any other tab or run in the background.

scripting

Used to scan the active page for image and video elements. The injected script reads media URLs from the DOM and returns them to the popup — it does not modify the page or read any user input.

downloads

Allows the extension to trigger file downloads through the browser's download manager, the same mechanism used when you click a normal download link.

host_permissions (<all_urls>)

Required so the extension can fetch image data (to build ZIP files) from any domain, matching wherever you happen to be browsing. No data from these requests is stored or forwarded.

Third-party services

Instaloader does not integrate with any third-party service, API, or SDK. The only network requests it makes are fetches for media files you explicitly choose to download, going directly to the same servers that served those files to your browser.

Children's privacy

This extension does not collect any data from anyone, including children under 13.

Changes to this policy

If this policy ever changes, the updated version will be published at the same URL with a revised date at the top. Given that no data is collected, meaningful changes are unlikely.
